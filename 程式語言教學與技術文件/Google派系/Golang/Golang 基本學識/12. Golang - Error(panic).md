# Golang - Error(panic)

Golang大多數的回傳函數都會自帶Error的回傳功能，但其實自己也可以做自己的錯誤中斷回傳功能，這就是panic。

直接來看下面的程式碼跟輸出：

```go
// region error
func ErrorPractice() {
	defer func() {
		fmt.Println("first")
		if err := recover(); err != nil {
			fmt.Println(err)
		}

		fmt.Println("end")
	}()
	f()
}
func f() {
	fmt.Println("test")
	panic("It's run panic")
	fmt.Println("test2")
}
```

```bash
test
first
It's run panic
end
```

- 必須先利用defer並搭配recover定義錯誤時的代碼，不然panic會直接出錯關閉程式(Sever這樣就很要不得阿)
- 當呼叫panic時，就會跳到defer上的函式作處理，傳送完錯誤處理後會繼續運行。
